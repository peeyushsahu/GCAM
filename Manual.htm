Manual for GCAM

System Requirements

Linux

- Java 1.5 or higher installed and in environmental path,

- R 2.5.0 or later and preferably installed on default location,

- Package Rserve installed in R, can be installed from CRAN library using install.packages ("Rserve"), for more information please visit http://rforge.net/Rserve/doc.html,

- Package gplots and scrime should also be installed with dependencies in R. 

Windows

- Java 1.5 or higher installed and in environmental path,

- R 2.5.0 or later installed

- Package gplots and scrime should be installed with dependencies in R. 

- Windows does not support proper Rserve functionality needed for the GCAM statistical analysis but it can be easily done manually as shown in later part.

R Packages

These packages can be installed by running below commands to your R terminal.

install.packages ("Rserve")

install.packages("gplots", dependencies = TRUE)

install.packages("scrime", dependencies = TRUE)

Starting GCAM

Linux

Type in command line "java -jar path/to/GCAM_jar_file_with_dependency". This will open a GUI for the analysis.

Windows

GCAM can be started by double click on the jar file. 

Abstract Fetching

GCAM takes input as a .CSV file, this consist one gene name per line (without spaces in the beginning or end). Example input file (Test_GeneNmaes.csv) can be found in GCAM_Project.

To upload, click on 'File' than browse and select the file of interest.

Additional query words are the filter for abstracts of interest, which can be added to the text box separated by comma "," e.g. Homo sapiens,mouse.

In the next step, we set the settings for abstracts download from Entrez server:

1) set the maximum number of abstracts to download for each gene, default is 3000;

2) how many abstracts should be fetched each second, default is 3 according to Entrez guideline which can be increased to 500 during weekend or between 9pm to 5am EST;

3) Add synonym provides with the option to consider synonym for each gene in Human or Mouse. After setting everything press the "Set" button.   

    In the next step, press "Fetch Abstracts" to proceed with downloading.

Named Entity Recognition

The next step after Abstract Fetching is "Named Entity Recognition".

To select the option for Named Entity Recognition(NER), please go to Analysis than click on the option. This will create proceed with NER and results will be stored in the directory named GCAM_OUTPUT which is automatically created in the user's home directory. After this analysis, result will be stored in two files: 1) count_table: contains occurrence of cell types per gene; 2) synonym: synonym for gene if option selected.

Enrichment Analysis

In the next step, statistical analysis for obtained count data will be performed. To proceed with the analysis please go to Analysis -> EnrichmentAnalysis. This will open a window with parameters to select for analysis.

 

In this window user can select multiple options:

- Merge synonym values is to combine count values for gene synonym into one. (Needed if you consider synonym)

- User can choose between two statistical tests.

- Threshold for p-value and enrichment of gene - cell type association.

The results will be saved in the output directory "GCAM_OUTPUT".

Enrichment Analysis for Windows OS

As mentioned before enrichment analysis does not work on windows properly. So users can do this with a little effort on R.

To analyze count table obtained by GCAM initial steps please install appropriate (mentioned above) packages in R.

In the second step, look for the R script written for analyzing the count data with name Palindrome.R. This can be found in the project folder under the directory named resource.

It is written as a function so it can be compiled easily. To compile you can copy the whole script and paste into R terminal and press Enter. After script is compiled next step to run it.

To run the script, write a command in R terminal as below (assign correct input type for parameters):

Palindrome (path, dirpath, Pthreshold, Ethreshold, synonym, test) 

path = (path to the GCAM output directory created in the home directory) e.g. "/home/user/GCAM_output";

dirpath = (path to the project folder where GCAM.jar is located) e.g. "/home/user/NetBeansProjects/GCAM";

Pthreshold = p-value threshold e.g. 0.001;

Ethreshold = enrichment threshold represents lower cutoff for celltype enrichment to be consider into analysis e.g. 0.2;

Synonym = 1: if synonym was considered else, 0: no synonyms were considered;

Test = statistical test binomial or fisher.

Pathway Analysis

As an optional analysis user can analyze the names of pathway user input gene are associated. To analyze go to Analyse -> PathwayAnalysis. The results will be saved in the output directory.
